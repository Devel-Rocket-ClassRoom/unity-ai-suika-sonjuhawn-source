# 수박게임 클론 - 게임 디자인 문서

> Unity 6 (6000.3.15f1) 기반 수박게임(Suika Game) 클론 프로젝트 설계서

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 물리 기반 머지(merge) 퍼즐 |
| **플랫폼** | PC (Windows), 모바일 확장 가능 |
| **엔진** | Unity 6 (6000.3.15f1) |
| **플레이 시간** | 1판당 3~10분 |
| **시점** | 2D 정면뷰 |
| **조작** | 마우스 좌우 이동 + 클릭으로 드롭 (모바일은 터치 스와이프) |

**한 줄 소개**: 통 안에 과일을 떨어뜨려 같은 과일끼리 합치고, 점점 더 큰 과일을 만들어 최종적으로 수박을 만드는 단순하지만 중독적인 퍼즐 게임.

**핵심 재미 포인트**
- 규칙은 1분 안에 이해 가능 (낮은 진입 장벽)
- 물리 엔진이 만드는 예측 불가능한 우연성
- "한 번만 더" 를 부르는 점수 갱신 욕구
- 시간 제한 없음 → 캐주얼/힐링 플레이 가능

## 2. 게임 시스템

### 2.1 드롭(Drop) 시스템
- 화면 상단에 현재 과일이 표시되고, 마우스/터치로 좌우 위치 조정
- 클릭(또는 탭) 시 해당 X좌표에서 과일이 자유낙하
- 드롭 후 일정 쿨다운(예: 0.5~1초) 동안은 다음 과일을 떨어뜨릴 수 없음
- 과일 가장자리가 통의 안쪽 벽에 닿는 위치까지 마우스 이동 가능 (반지름 기반 클램프)

### 2.2 머지(Merge) 시스템
- **같은 단계의 과일 2개가 충돌**하면 한 단계 위 과일로 합쳐짐
- 새 과일의 생성 위치 = 두 과일의 중간 지점
- 새 과일에는 약간의 위쪽 임펄스를 줘서 자연스러운 연쇄 머지 유도
- 머지 시점에 점수 가산 + 이펙트(파티클) + 사운드 재생

### 2.3 물리 시스템
- Unity `Rigidbody2D` + `CircleCollider2D` 사용
- 중력, 마찰, 반발(bounciness) 설정으로 통통 튀는 느낌
- 모든 과일은 원형 충돌체 (계산이 단순하고 굴러가는 느낌이 좋음)

### 2.4 다음 과일 예고 시스템
- 화면 우상단(또는 좌상단)에 "Next" 영역
- 미리 다음에 떨어질 과일을 보여줘서 전략 수립 가능
- 미리보기 크기는 과일 반지름에 비례 (작은 과일은 작게, 큰 과일은 크게)

### 2.5 드롭 풀(Pool) 시스템
- 자연 드롭으로 등장하는 과일은 **작은 단계 5종으로 제한** (체리 ~ 감)
- 그 이상 단계 과일은 오직 머지로만 생성됨
- 등장 확률은 균등하게 하거나, 작은 과일일수록 약간 더 자주 나오게 조정

## 3. 과일 단계 / 점수 / 승패 규칙

### 3.1 과일 진화 11단계

| 단계 | 과일 | 크기(상대) | 머지 점수 | 자연 드롭 |
|------|------|-----------|----------|-----------|
| 1 | 체리 🍒 | 매우 작음 | 1 | O |
| 2 | 딸기 🍓 | 작음 | 3 | O |
| 3 | 포도 🍇 | 작음 | 6 | O |
| 4 | 데코폰(귤) 🍊 | 중소 | 10 | O |
| 5 | 감 🟠 | 중소 | 15 | O |
| 6 | 사과 🍎 | 중간 | 21 | X |
| 7 | 배 🍐 | 중간 | 28 | X |
| 8 | 복숭아 🍑 | 큼 | 36 | X |
| 9 | 파인애플 🍍 | 큼 | 45 | X |
| 10 | 멜론 🍈 | 매우 큼 | 55 | X |
| 11 | 수박 🍉 | 최대 | 66 | X |

> 점수 공식은 **삼각수(triangular number)**: `n*(n+1)/2` (n = 단계 번호). 자유롭게 조정 가능.

### 3.2 점수 시스템
- 머지가 일어날 때마다 **새로 생성된 과일의 점수**가 총점에 가산
- 큰 과일을 만들수록 누적 점수가 기하급수적으로 늘어남
- 콤보(연쇄 머지)는 별도 보너스로 줄지 옵션

### 3.3 게임 오버 조건
- 통 상단에 **빨간 데드라인**(경고선)이 있음
- 과일이 데드라인 위로 올라온 상태가 **일정 시간(예: 2~3초) 지속**되면 게임 오버
- 막 떨어뜨린 직후의 일시적인 솟음은 무시 (그렇지 않으면 너무 짜증남)

### 3.4 승리/엔딩
- **명시적 "승리" 조건 없음** — 끝없이 점수를 갱신해가는 스코어 어택 게임
- **수박을 만든 시점**이 사실상의 메인 목표 달성 = 1차 엔딩
- 수박 2개 만들기("2수박")는 고수의 상징
- 수박 + 수박 머지 시: 두 과일이 사라지며 큰 보너스 점수 (디자인 선택)

## 4. 화면 UI

### 4.1 메인 게임 화면 구성

```
┌─────────────────────────────────────┐
│  점수: 1234       [Next: 🍓]        │  ← 상단 HUD
│                                     │
│  ← 현재 과일 🍒 →                    │  ← 드롭 위치 표시
│  - - - - - - - - - - - - - - -     │  ← 빨간 데드라인
│  │                              │   │
│  │                              │   │  ← 게임 통(컨테이너)
│  │       🍎  🍇                 │   │
│  │   🍓 🍒🍒🍑 🍐               │   │
│  │  🍓🍇🍎🍑🍑🍓🍇             │   │
│  └─────────────────────────────┘   │
│            [⏸ 일시정지]              │  ← 하단
└─────────────────────────────────────┘
```

### 4.2 화면별 UI 구성

**메인 메뉴 화면**
- 게임 타이틀 로고
- "Play" 버튼
- "최고 기록" 표시
- 설정 / 도움말 아이콘

**인게임 화면**
- 상단 좌측: 현재 점수 (큰 폰트)
- 상단 우측: 다음 과일 미리보기 (크기는 반지름 비례)
- 중앙: 게임 통, 데드라인, 현재 떨어뜨릴 과일
- 하단: 일시정지 / 다시시작 버튼 (작게)
- 우상단 모서리: 과일 도감/진화 차트 (옵션)

**게임 오버 화면**
- "Game Over" 텍스트
- 최종 점수
- 최고 기록 갱신 시 ✨ 강조
- "다시하기" / "메인으로" 버튼
- (옵션) 최대 도달 과일 표시: "당신은 🍐 까지 만들었어요!"

**일시정지 화면**
- 반투명 오버레이
- 재개 / 다시시작 / 메인으로 버튼

### 4.3 UI 디자인 원칙
- **미니멀** — 화면 중앙의 게임 플레이를 방해하지 않음
- **큰 터치 영역** — 모바일 호환성 고려
- **즉각적 피드백** — 점수 증가 시 숫자 팝업, 머지 시 화면 효과
- **한국어 우선**, 영어 토글 가능 (글로벌 확장 고려)

## 5. 톤과 분위기

### 5.1 비주얼 톤
- **밝고 귀여운 카툰풍** — 어린이부터 어른까지 편안하게 즐길 수 있는 분위기
- **부드러운 일러스트** — 과일은 사실적이지 않고 동글동글하고 사랑스럽게
- **표정/디테일** — 큰 과일일수록 살짝 다른 표정이나 반짝임 효과 (희소성 표현)

### 5.2 색감
- **파스텔 + 비비드의 중간** — 너무 채도가 낮으면 심심하고, 너무 진하면 피로
- **배경**: 따뜻한 베이지/연한 노랑 또는 그라데이션 (테이블 위 느낌)
- **통**: 나무 또는 유리 질감 — 친근한 느낌
- **데드라인**: 명확한 빨간색이지만 부드러운 톤 (위협적이지 않게)

### 5.3 사운드 디자인
- **BGM**: 잔잔하고 반복 부담 없는 어쿠스틱/피아노 멜로디, 음량 작게
- **머지 SFX**: 작은 과일은 가볍게 "퐁/통", 큰 과일은 더 묵직하게 "팡!"
- **드롭 SFX**: 부드러운 "툭" 소리
- **수박 머지 SFX**: 특별한 환호/팡파레 효과
- **게임 오버 SFX**: 슬프지 않고 살짝 코믹하게 (다시 하고 싶게)

### 5.4 전체 무드 키워드
- 힐링, 캐주얼, 무해함, 중독성, 만족감, 귀여움
- **회피 무드**: 긴장, 압박, 스트레스, 폭력적, 시끄러움

### 5.5 핵심 감정 곡선
```
드롭 (집중) → 머지 (쾌감!) → 연쇄 (대박!) → 위기 (긴장) → 게임오버 (아쉬움) → 다시! (의지)
```

## 부록: 개발 단계 제안

1. **MVP (Minimum Viable Product)**
   - 과일 5종, 드롭/머지/게임오버만 작동
   - UI는 점수 표시 + 게임오버 텍스트 정도

2. **핵심 기능 완성**
   - 11종 과일 전체 적용
   - Next 과일 표시, 사운드, 이펙트
   - 메인 메뉴 / 게임오버 화면

3. **폴리싱**
   - 파티클, 트레일, 카메라 셰이크
   - 최고 점수 저장 (PlayerPrefs)
   - 도감 / 통계 화면

4. **확장 (선택)**
   - 모바일 빌드
   - 캐릭터 스킨, 통 테마
   - 일일 챌린지, 리더보드
